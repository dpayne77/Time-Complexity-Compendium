About the Course


The Time Complexity Compendium is aimed at late-high school or early undergraduate Computer Science students who already have some programming experience (such as Programming Fundamentals 1 or similar). The course is designed to give students an introduction to Time Complexity and and Big O Notation. Most importantly, the course teaches students how to analyze the time complexity of algorithms under various circumstances, such as loops, recursion, and data structures.

Outline


Key Content


This mini-course will start by covering the mathematical functions and the growth rates that underlie time complexity in software engineering before moving on to covering the fundamentals of Big-O notation. Students will then learn to evaluate code snippets involving loops, recursion, data structures, and special cases using Big-O notation for future applications in their careers.


Learning Objectives


  • Understand the mathematical functions and growth rates that underlie time complexity in software engineering.
  • Understand and apply the fundamentals of Big-O notation.
  • Analyze code snippets involving loops, recursion, data structures, and special cases using Big-O notation.
  • Compare and evaluate time complexity of different code snippets to determine which is more efficient.

For more information about how these objectives align with national standards, see the "Course Schedule" section.


Structure of Course


The mini-course is a Self-paced course. This means that students can complete the modules at their own pace according to their understanding of each topic. Since modules build on each previous topic, it is highly recommended that students take and pass the summative evaluation assessment at the end of each module before moving on to the next, even though all modules are unlocked from the start. Because this course is online, students can always reference and review previous modules if they fail to understand a certain topic.

We recommend students spend approximately 1 hour per day on this course for approximately 2 school weeks, doing about 1 module per day. However, students can complete the course at their own pace.

There are 7 modules, each intended to take approximately 45 minutes, including the summative evaluation assessment. The modules will be a mix of instructional content via paragraphs, diagrams, visuals, and code examples. Interspersed between the instructional content will be mini-activities that check the studentâ€™s understanding of the content that came before it. These activities give students the opportunity to think critically about content and reinforce learning by allowing them to apply what they have learned. At the end of each module, there will be a summative evaluation assessment consisting of questions related to the module.


Course Resources


The course and all of its modules are available online, completely free, at the website cise.ufl.edu/~dpayne1/TimeComplexity/

You will not need to download extra software for this course, nor do you need to create an account to start learning. All you need is an internet connection and a browser.

You can access the course on any device that has a web browser, including desktops, laptops, tablets, and smartphones. The course is designed to be responsive, meaning that it will automatically adjust to the size of your screen. This means that you can take the course on any device, no matter the size of the screen.

It is recommended that students are familiar with the C++ programming language, as the code snippets in this course will be written in C++. The course will not delve deep into C++ syntax, but students should know the syntax for variable declarations, for loops, while loops, and recursive functions for this course. For students unfamiliar with C++, the website w3schools.com/cpp/ would be a good place to start learning this syntax.

Grading & Assessments


There are 7 summative evaluation assessments consisting of multiple choice, true/false, select all that apply, and short answer questions at the end of each module. These are weighted at 60% of a student's final grade and are individual assessments. The remaining 40% are activities that are also completed individually. In every module, there is an activity or multiple activities that students can complete to further reinforce their understanding of the content.
